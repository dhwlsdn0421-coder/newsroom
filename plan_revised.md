# 프로젝트 계획서: YouTube Info Viewer (수정 초안)

PyQt5와 yt-dlp를 활용하여 유튜브 영상의 정보를 조회하는 데스크톱 애플리케이션입니다.

## 1. 개요
- **목적**: 유튜브 URL을 통해 영상의 메타데이터(썸네일, 제목, 조회수)를 쉽고 빠르게 확인.
- **주요 도구**:
    - **Python 3.10+** (권장; 3.14 환경에서 호환 여부는 구현 시 검증)
    - **PyQt5**: GUI 프레임워크
    - **yt-dlp**: 유튜브 데이터 추출 라이브러리
    - **requests**: 썸네일 이미지 다운로드를 위한 HTTP 라이브러리

> **참고**: YouTube 정책·yt-dlp 변경 시 동작에 영향이 있을 수 있음. 장기적으로는 Phase 2에서 YouTube Data API 등 대안 검토 가능.

---

## 2. 주요 기능 (Phase 1)
- [ ] **유튜브 URL 입력**: 사용자가 분석할 영상의 주소를 입력하는 필드.
- [ ] **정보 조회 버튼**: 클릭 시 `yt-dlp`를 실행하여 데이터를 가져오는 트리거.
- [ ] **결과 표시 영역**:
    - **썸네일**: 이미지 레이블에 시각적으로 표시. 로드 실패 시 placeholder 또는 에러 아이콘 표시.
    - **제목**: 영상의 제목 텍스트 표시.
    - **조회수**: 천 단위 구분 기호(,)를 포함한 조회수 표시.

---

## 3. URL 유효성 검사 기준
- **허용 형식**: `youtube.com/watch?v=...`, `youtu.be/...`, Shorts (`youtube.com/shorts/...`) 등 유튜브 영상 URL.
- **검사 시점**: 조회 버튼 클릭 시.
- **거부 시**: 유효하지 않은 URL이면 즉시 안내 메시지 표시 후 조회하지 않음.

---

## 4. UI/UX 설계
- **레이아웃**: `QVBoxLayout` (수직 배치) 기반
- **위젯 구성**:
    - **상단 (입력부)**: `QLineEdit` (URL 입력), `QPushButton` (조회)
    - **중앙 (이미지부)**: `QLabel` (썸네일 출력, 고정 크기 또는 비율 유지). 썸네일 없음/로드 실패 시 placeholder 이미지 또는 "이미지를 불러올 수 없습니다" 등 안내 표시.
    - **하단 (정보부)**:
        - `QLabel` (제목: 볼드체, 자동 줄바꿈)
        - `QLabel` (조회수: 색상 구분)

---

## 5. 로직 상세 (Workflow)
1. **입력 처리**: 사용자가 입력한 URL을 위 "3. URL 유효성 검사 기준"으로 검사.
2. **백그라운드 실행**: `yt-dlp`·`requests` 호출은 **메인 UI 스레드가 블로킹되지 않도록** 워커 스레드(또는 QThread 등)에서 수행. 완료 시 시그널/콜백으로 메인 스레드에서 UI만 갱신.
3. **데이터 추출 (yt-dlp)**:
   - `YoutubeDL` 객체를 사용하여 `extract_info` 호출.
   - `video_id`, `title`, `view_count`, `thumbnail_url` 추출.
4. **이미지 로드**:
   - `requests`를 통해 썸네일 URL의 이미지 데이터를 바이너리로 획득.
   - `QPixmap`을 사용하여 레이블에 셋팅. 실패 시 placeholder 또는 에러 표시.
5. **결과 업데이트**: 추출된 텍스트 정보를 UI 위젯에 반영 (메인 스레드에서만).
6. **예외 처리**: 아래 구분에 따라 `QMessageBox` 등으로 사용자에게 명확히 안내.
   - **존재하지 않는 영상**: "영상을 찾을 수 없습니다."
   - **네트워크 오류**: "네트워크 연결을 확인해 주세요."
   - **나이 제한·비공개 등**: "이 영상은 정보를 가져올 수 없습니다." (또는 yt-dlp 메시지 정제)
   - **잘못된 URL**: "유효한 유튜브 URL을 입력해 주세요."
