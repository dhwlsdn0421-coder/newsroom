# RSS 수집 기능 명세서 (보완판)

> AI Studio 초안 검토를 반영하여 요구사항·예외·출력 형식·구조를 명확히 한 명세입니다.  
> 구현 전 이 문서를 기준으로 합니다.

---

## 1. 개요

- **목적**: RSS 피드 URL로부터 최신 기사 목록을 수집하여 일관된 형식으로 반환한다.
- **주요 의존성**: `feedparser` (버전 범위는 2.2절 참조).

---

## 2. 요구사항

### 2.1 입력

| 항목 | 타입 | 제약 | 비고 |
|------|------|------|------|
| `feed_url` | `str` | 비어 있지 않음. 앞뒤 공백 제거 후 검사. `http://` 또는 `https://`로 시작. | 리다이렉트·쿼리스트링 허용. |
| `limit` | `int` | 1 이상, 100 이하. 범위 밖이면 각각 1, 100으로 보정. | 수집할 **기사 개수 상한**. |

- **검증 시점**: 함수 진입 직후. 위 제약 위반 시 명시된 동작(3.2절) 따름.

### 2.2 출력 (반환 스키마)

반환값은 **기사 정보(dict)의 리스트**이다. 각 dict는 아래 키를 가진다.

| 키 | 타입 | 필수 | 설명 |
|----|------|------|------|
| `title` | `str` | O | 기사 제목. 없으면 빈 문자열. |
| `link` | `str` | O | 기사 URL. 없으면 빈 문자열. |
| `published` | `str` | X | 발행일 시각. ISO 8601 형식 문자열 권장; 파싱 실패·미제공 시 `None`. |
| `summary` | `str` | X | 요약/본문 일부. HTML 태그는 제거한 순수 텍스트. 없으면 `None`. |

- **정렬**: 수집된 항목을 **시간 역순(최신 먼저)**으로 정렬한 뒤, 앞에서부터 `limit`개만 반환.
- **정렬 기준**: `published`(또는 이에 대응하는 feedparser 필드) 우선; 없거나 동일하면 피드 원래 순서 유지.
- **중복**: 동일 `link`를 가진 항목은 **처음 등장한 것만** 포함(중복 제거).

### 2.3 의존성

- **feedparser**: `>=6.0.0, <7.0.0` (호환성·보안 고려해 범위 명시).
- **Python**: 3.10+ (프로젝트 정책에 맞춤).

---

## 3. 예외 및 오류 처리

### 3.1 예외 분류와 동작

| 구분 | 조건 | 동작 |
|------|------|------|
| **입력 오류** | `feed_url`/`limit` 검증 실패 | 예외 발생. `ValueError` 및 메시지로 사유 전달. |
| **네트워크 오류** | 연결 실패, 타임아웃, DNS 실패, HTTP 4xx/5xx | 예외 발생. 가능하면 `requests`/`urllib` 계열 예외 유지하거나, 래핑 시 `ConnectionError`/`TimeoutError` 등 구체적 타입 사용. |
| **파싱 오류** | feedparser `bozo==True` 이며 entries를 쓸 수 없음 | 예외 발생. 예: `ValueError("RSS 파싱 실패: ...")`. |
| **부분 성공** | `bozo==True`이지만 `entries`는 존재 | entries만 사용해 정상 진행. 파싱 경고는 로깅만. |
| **데이터 구조** | `entries` 없음 또는 리스트 아님 | 빈 리스트 반환. 예외 없음. |
| **인코딩 오류** | 응답 body 디코딩 실패 | 예외 발생. `UnicodeDecodeError` 또는 래핑한 `ValueError`. |

- **로깅**: 네트워크/파싱/인코딩 오류 발생 시 원인을 로그에 기록하는 것을 권장(요구사항으로 명시 가능).

### 3.2 입력 검증 상세

- `feed_url`이 `None`이거나 문자열이 아니면 → `ValueError`.
- 앞뒤 공백 제거 후 빈 문자열이면 → `ValueError`.
- `http://` 또는 `https://`로 시작하지 않으면 → `ValueError`.
- `limit`이 정수가 아니거나 1 미만이면 → 1로 보정.
- `limit`이 100 초과면 → 100으로 보정.

## 4. 구현 구조 (권장 단계)

아래 순서로 단계를 나누어 구현하면 모호함과 초보자 막힘이 줄어든다.

1. **입력 검증**  
   `feed_url`, `limit` 검사 및 보정. 위반 시 `ValueError`.

2. **RSS 가져오기·파싱**  
   - 가능하면 “URL에서 raw 데이터 가져오기”(타임아웃·크기 제한 적용)와 “feedparser로 파싱”을 분리.
   - 타임아웃: 15초 권장. 최대 응답 크기: 5MB 등 상한 권장(선택).

3. **파싱 결과 검증**  
   - `bozo`, `status`, `entries` 존재·타입 확인.
   - “정상 아님”일 때의 규칙은 3.1절에 따름(예외 vs 빈 리스트).

4. **entries → 기사 dict 변환**  
   - feedparser 항목의 `.title`, `.link`, `.published`, `.summary` 등에서 위 스키마로 매핑.
   - `summary`에 HTML 포함 시 태그 제거 후 저장.
   - 필드 없을 때: 2.2절 표의 기본값(빈 문자열/`None`) 적용.

5. **정렬·중복 제거·limit**  
   - `published`(또는 대체 필드) 기준 역순 정렬.
   - 동일 `link` 중복 제거(첫 항목만 유지).
   - 그 다음 앞에서부터 `limit`개 슬라이스.

6. **반환**  
   - 성공: 기사 dict 리스트 반환.
   - 실패: 3.1절에 따라 예외 또는 빈 리스트.

---

## 5. 주의 사항 (초보자 막힘 방지)

- **feedparser 구조**: `feed.entries`는 리스트이며, 각 항목은 `.title`, `.link`, `.published`, `.summary` 등 속성을 가짐. 필드명은 피드에 따라 다를 수 있으므로 문서 및 실제 피드로 확인.
- **날짜**: `published`는 문자열이거나 파싱된 객체일 수 있음. ISO 8601 문자열로 통일해 반환할지, 파싱 실패 시 `None` 처리할지 이 명세대로 일관 적용.
- **HTML in summary**: 요약에 HTML이 포함되면 “태그 제거 후 순수 텍스트만 저장”으로 통일.
- **예외 범위**: `except Exception`만 쓰지 말고, 3.1절에 따라 네트워크/파싱/입력/인코딩별로 구체적 예외 처리.
- **테스트**: 실제 URL 없이 테스트하려면 “문자열 XML → feedparser 파싱 → entries 처리” 단계를 분리해 두고, 문자열/파일 입력을 주입할 수 있게 하면 유리함.
